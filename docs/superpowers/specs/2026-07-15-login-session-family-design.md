# 登录会话族展示设计

## 问题

刷新令牌每轮换一次都会创建一条 `refresh_sessions` 记录。当前个人中心直接返回这些令牌记录，且只依据 `revokedAt` 显示状态；已被轮换消耗的记录因此被错误标为“有效”，多次刷新页面后会造成大量“有效会话”。

## 决策

以 `familyId` 作为一个登录会话的稳定标识。后端只向个人中心返回每个会话族的最新令牌记录，并由该记录计算会话状态：未撤销、未消耗且未到期为有效；撤销为已退出；到期为已过期。轮换链中的旧令牌不再出现在会话列表。

## 范围

- 保留每次 Refresh Token 轮换创建新记录的安全模型与重放检测。
- 会话接口按会话族聚合，只返回仍有效的会话，不泄露刷新令牌或其哈希。
- 个人中心移除手动“刷新会话”按钮；应用恢复和请求失败时仍按既有逻辑刷新令牌。
- 页面改为展示会话状态、首次登录时间、最近活动时间和到期时间；不再展示原始数据库会话 ID。
- “退出当前登录”和“退出全部设备”继续保留。

## 非目标

- 本期不采集浏览器、IP 或设备名称，避免在没有可信采集数据时展示误导性设备信息。
- 本期不清理历史令牌审计数据。
