# 简历反馈设计

## 目标

为公开简历增加低阻力的匿名反馈能力：访客可在不登录的情况下即时表达阅读感受，或单独提交评分与文字建议；超级管理员可在现有“简历管理”页面集中查看、筛选和处理反馈。

## 交互

### 1. 阅读感受

- 桌面端在简历卡片左下方的卡片外侧显示 5 个表情。
- 点击表情立即提交，不显示表单或二次确认。
- 成功后原位置替换为“感谢你的反馈”，本次浏览不允许重复提交表情。
- 移动端将该入口放到简历正文末尾的紧凑横向区域，保证不遮挡内容和系统手势区域。

### 2. 评分与建议

- 桌面端在简历卡片右上方的卡片外侧显示轻量文字入口“留下评分或建议”。
- 点击入口打开模态框：1–5 分评分必填，建议文本选填且最长 1,000 字。
- 提交成功后关闭模态框，并将入口文字替换为“感谢你的反馈”。
- 提交失败时保留评分和文本，显示明确错误信息；网络错误不得丢失用户输入。
- 模态框支持键盘焦点管理、Esc 关闭和移动端安全区域；建议文本可选择、复制和粘贴。

## 数据与接口

新增 `resume_feedbacks` 表。每条记录属于以下一种类型：

| 类型 | 必填字段 | 可选字段 |
| --- | --- | --- |
| `REACTION` | `reaction`（五档表情） | 无 |
| `REVIEW` | `rating`（1–5） | `suggestion` |

公共接口：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/api/v1/resume/feedback/reactions` | 立即提交一条表情反馈 |
| `POST` | `/api/v1/resume/feedback/reviews` | 提交评分及可选建议 |

管理接口均要求 `resume:manage`：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/v1/resume/feedback` | 分页查询，支持类型、处理状态和评分筛选 |
| `PATCH` | `/api/v1/resume/feedback/{id}/status` | 更新为“已读”或“已归档” |

每条反馈保存提交 IP、浏览器 User-Agent、创建时间、类型、表情/评分、建议和处理状态。IP 仅对有 `resume:manage` 权限的后台可见。服务端按 IP 和反馈类型限制：表情每 24 小时 1 次，评分建议每 15 分钟 1 次；命中限制时返回可读错误信息。

## 后台管理

现有“内容管理 → 简历管理”保留简历编辑功能，新增“反馈”页签：

- 概览显示评分平均值、评分数量和表情分布。
- 列表显示反馈类型、评分/表情、建议摘要、IP、提交时间和状态。
- 支持按类型、状态和评分筛选；建议可展开完整查看及复制。
- 可将未读反馈标记为已读或归档；不提供公开修改或删除反馈的能力。

## 安全与约束

- 反馈接口公开但仅接受经过 Bean Validation 验证的 DTO；建议按纯文本存储和渲染，禁止 HTML。
- `resume:manage` 仍仅由 `SUPER_ADMIN` 获得；不新增公开管理权限。
- 反馈独立于单份 `resume_documents` JSON，不影响简历的 SSR 读取和编辑版本号。
- 前端页面的公开提交使用 Nuxt runtime config 中已有的简历 API 基地址。

## 验收

1. 访客点击表情后立即成功，原入口显示感谢，并写入一条 `REACTION` 记录。
2. 访客能在模态框中提交 1–5 分和可选建议；提交成功后右上入口显示感谢。
3. 无评分的评分建议、非法表情、超长建议和频率过高请求均被拒绝并显示可读提示。
4. 超级管理员可在后台筛选和处理两种反馈，普通管理员无法访问管理接口或页面。
5. 移动端两种入口均可使用，且不会遮挡简历正文或底部系统区域。
