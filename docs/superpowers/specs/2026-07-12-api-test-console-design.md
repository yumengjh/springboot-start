# API 测试台改造设计

## 目标

将根目录 `api-test.html` 从手工填写 ID 和权限编码的接口表单，改为可直接测试管理员、用户和 RBAC 能力的单页操作台。

## 页面结构

- 登录区：默认填写本地 `.env` 示例的 `admin`，密码仍由使用者输入，避免将密码写入仓库。
- 用户管理区：加载 `/api/v1/admin/users` 为表格，显示总用户数、用户名、显示名、状态与 ID；点击行选择用户，并可更新该用户状态、分配或移除角色。
- RBAC 区：加载 `/api/v1/rbac/roles`。左侧选择角色，右侧按模块分组展示全部已知权限；勾选权限后调用现有授予/移除接口，并刷新角色数据。
- 当前用户与运行时配置区：保留现有自助资料、改密、运行时配置接口。
- 响应区：每次请求都显示方法、路径、状态码与 JSON；表格操作成功后自动刷新相关数据。

## 边界

- 不新增后端 API。
- “查看用户”使用当前的管理员用户列表；当前后端没有单用户详情接口。
- 管理员仅能修改现有接口支持的用户状态和角色，不能修改其他用户的显示名或密码。
- 不在 HTML 中保存默认管理员密码或令牌；令牌仅驻留当前页面内存。

## 验收

使用超级管理员登录后，能看见用户总数、选择用户、变更状态与角色；能查看每个角色拥有的权限、清晰地勾选更新权限；所有操作的请求结果可见。
